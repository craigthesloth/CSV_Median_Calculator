# CSV Median Calculator

Консольное приложение для инкрементального расчёта медианы цен из CSV-файлов с биржевыми данными.

## Описание

Приложение читает один или несколько CSV-файлов из указанной директории, содержащих историю торгов, объединяет все записи, сортирует по временной метке `receive_ts` и последовательно вычисляет медиану цен. Каждый раз, когда медиана изменяется, в выходной файл записывается строка с временной меткой и новым значением медианы.

### Ключевые особенности
- **Точная инкрементальная медиана** (структура «две кучи») – соответствует определению медианы как среднего двух центральных для чётного количества элементов.
- **Многопоточное чтение файлов** – ускорение обработки за счёт параллельного чтения.
- **Логирование в консоль и файл** с ротацией (папка `resources/logs`).
- **Конфигурация в формате TOML** – гибкая настройка входных/выходных путей и фильтров.
- **Корректная обработка ошибок** – приложение не падает, все исключения логируются с указанием контекста.
- **Модульное тестирование** (опционально) – набор тестов для проверки корректности расчёта медианы и обработки ошибок.

## Требования

- Компилятор с поддержкой **C++23** (GCC 14+, Clang 18+, MSVC 2022 17.6+)
- **CMake** 3.23 или выше
- Интернет для скачивания зависимостей (используется `FetchContent`)

Зависимости (автоматически подгружаются):
- Boost (filesystem, program_options, tokenizer) – для работы с файловой системой и парсинга CSV
- toml++ – парсинг TOML-конфигурации
- spdlog – логирование
- (опционально) Catch2 – для модульных тестов

## Сборка проекта

1. Клонируйте репозиторий:
   ```bash
    git clone https://github.com/craigthesloth/CSV_Median_Calculator
    cd CSV_Median_Calculator
   ```

2. Создайте директорию для сборки и запустите CMake:
   ```bash
   mkdir build && cd build
   cmake .. -DCMAKE_BUILD_TYPE=Release
   cmake --build . --config Release
   ```

3. Исполняемый файл будет создан в подпапке `bin` или непосредственно в `build` (в зависимости от генератора).

## Структура ресурсов

После сборки рекомендуется разместить ресурсы следующим образом (или использовать существующую структуру в корне проекта):

```
.
├── resources/
│   ├── configs/               # конфигурационные файлы TOML
│   │   └── config.toml
│   ├── data/
│   │   ├── input/              # входные CSV-файлы
│   │   └── output/              # выходные результаты (создаётся автоматически)
│   └── logs/                    # файлы логов с ротацией
├── CSV_Median_Calculator        # исполняемый файл (название может отличаться)
└── ...
```

### При запуске приложение:
- ищет конфигурационный файл сначала в `resources/configs/config.toml`, затем рядом с исполняемым файлом;
- создаёт выходную директорию, если она не существует;
- пишет логи в `resources/logs/app.log` (ротация 5 МБ, 3 файла).

## Запуск приложения

### С явным указанием конфигурационного файла
```bash
./CSV_Median_Calculator --config /путь/к/config.toml
./CSV_Median_Calculator -cfg config.toml
```

### Если аргументы не указаны
```bash
./CSV_Median_Calculator
```
Используется `resources/configs/config.toml` (или `config.toml` рядом с исполняемым файлом).

### Поддерживаемые аргументы командной строки
- `-h`, `--help` – показать справку.
- `-c`, `--config` – путь к конфигурационному файлу.
- `--cfg` – альтернативный флаг для указания конфига (синоним).

## Конфигурационный файл (TOML)

Пример `config.toml`:

```toml
[main]
# Обязательный параметр: путь к директории с входными CSV-файлами
input = "resources/data/input"

# Опционально: путь для сохранения результатов
# Если не указан, используется "resources/data/output"
output = "resources/data/output"

# Опционально: список масок для фильтрации файлов
# Файл будет обработан, если его имя содержит хотя бы одну из масок.
# Пустой список (или отсутствие параметра) означает "все .csv файлы".
filename_mask = ["level", "trade"]
```

## Входные данные

Приложение читает файлы с расширением `.csv`. Ожидается, что каждая строка (кроме заголовка) содержит поля, разделённые точкой с запятой (`;`). Для расчёта медианы используются два поля:
- `receive_ts` (первое поле) – временная метка в микросекундах (`uint64`);
- `price` (третье поле) – цена (`double`).

Строки с недостаточным количеством полей или некорректными числами пропускаются с предупреждением (указывается файл и номер строки).

## Выходные данные

Результат сохраняется в файл `result.csv` внутри выходной директории. Формат:

```
receive_ts;price_median
1716810808663260;68480.10000000
1716810809314641;68480.05000000
...
```

Каждая строка соответствует моменту, когда медиана всех обработанных на данный момент цен **изменилась**.

## Логирование

- Все информационные сообщения, предупреждения и ошибки выводятся в консоль (цветной вывод) и дублируются в файл `resources/logs/app.log`.
- Файловый лог ротируется по достижении 5 МБ, сохраняется до трёх архивных копий.
- Для форматирования сообщений используется `std::format` (C++23).

## Тестирование

При сборке с опцией `-DBUILD_TESTING=ON` будут скомпилированы модульные тесты на базе Catch2. Для их запуска выполните:

```bash
cd build
ctest -C Release --output-on-failure
```

Тесты проверяют:
- точность расчёта медианы на различных наборах данных (включая чётное/нечётное количество);
- обработку некорректных строк CSV;
- загрузку конфигурации.

## Примеры работы

### Пример 1. Базовая обработка

Входной файл `trade.csv`:
```
receive_ts;exchange_ts;price;quantity;side
1000;900;100.0;1.0;bid
2000;1900;101.0;2.0;ask
2000;1900;102.0;1.5;bid
3000;2900;103.0;3.0;ask
```

Выходной `result.csv`:
```
receive_ts;price_median
1000;100.0
2000;100.5
2000;101.0
3000;101.5
```

### Пример 2. Обработка пустого файла

При наличии пустого файла `level.csv` приложение выведет предупреждение, но продолжит работу с остальными файлами.

### Пример 3. Отсутствие входных файлов

Если в указанной директории нет подходящих CSV-файлов, приложение завершится с ошибкой и кодом возврата 1.
